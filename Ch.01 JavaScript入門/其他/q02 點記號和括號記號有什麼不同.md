# 點記號 (person.name)、括號記號 (person['age'])有什麼不同

在 JavaScript 中，點記號（dot notation）和括號記號（bracket notation）都是用來訪問對象屬性的方式，但它們有一些重要的不同點：


## 點記號 (Dot Notation)
- 語法：`object.property`
- 使用情境：
  - 當屬性名稱是有效的變量名稱時（例如，不包含空格、特殊字符等）。
  - 更易於閱讀和寫作。
- 範例：
```JavaScript
const person = {
    name: "Alice",
    age: 30
};

console.log(person.name); // 輸出: Alice

```

## 括號記號 (Bracket Notation)
- 語法：`object['property']`
- 使用情境：
  - 當屬性名稱是動態的或包含特殊字符（例如空格、破折號等）時。
  - 當屬性名稱是變量時。
- 範例：
```JavaScript
const person = {
    'first name': "Alice",
    age: 30
};

console.log(person['first name']); // 輸出: Alice
```
## 總結
- 可讀性：點記號通常更易於閱讀。
- 靈活性：括號記號更靈活，適用於特殊情況。
- 動態屬性：如果你需要根據變量來訪問屬性，必須使用括號記號。
- 這樣的區別使得在不同情境下選擇合適的記號非常重要。
